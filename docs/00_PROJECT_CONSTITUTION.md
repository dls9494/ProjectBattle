# PROJECT BATTLE CONSTITUTION

**Version:** 1.0.0
**Status:** Active
**Owner:** David
**Project Codename:** Project Battle

---

# Purpose

Project Battle is a personal mission intelligence system.

It is designed to transform daily health data into evidence-based intelligence.

The system exists for one purpose:

> **"The app doesn't tell me what to do. It tells me what my data says."**

---

# Vision

Project Battle is not a fitness application.

Project Battle is not a calorie tracker.

Project Battle is not a motivational app.

Project Battle is a personal intelligence system that:

- Observes
- Measures
- Explains
- Projects
- Remembers

using only user-provided data and deterministic calculations.

---

# Core Philosophy

Data over opinions.

Evidence over assumptions.

Transparency over black-box decisions.

Consistency over complexity.

Facts over motivation.

# Non-Negotiable Principles

## 1. Facts First

The application stores facts, not opinions.

Examples of facts:

- Water intake
- Calories
- Protein
- Sleep
- Weight
- Steps
- Walking duration
- Walking calories

Everything else is derived.

---

## 2. Every Insight Requires Evidence

No conclusion may be shown unless it can be supported by user data.

Every intelligence report must include:

- Observation
- Why it matters
- Evidence
- Confidence
- Recommendation (optional)

---

## 3. The User Owns the Decision

Project Battle never tells the user what they must do.

It presents evidence and allows the user to decide.

---

## 4. Compare Only Against History

The system compares the user only against their own historical data.

No comparison against:
- Population averages
- Other users
- Generic fitness standards

---

## 5. If Evidence Is Weak, Say So

If there is insufficient data, the system must explicitly state that a reliable conclusion cannot be made.

# Engineering Principles

## Store Facts, Generate Intelligence

Raw user data is the source of truth.

The application stores:

- Facts
- Events
- Measurements

The application never stores:

- AI conclusions
- Recommendations
- Intelligence reports

These are generated dynamically from the latest available data.

---

## Offline First

Project Battle must function completely offline.

Internet access is optional and only required for future AI integrations.

---

## Local First

All user data belongs to the user.

Version 1 stores everything locally on the device.

---

## Deterministic Calculations

The following must never depend on AI:

- BMI
- BMR
- Maintenance Calories
- Daily Deficit
- Mission Progress
- Confidence Score
- Reliability Score
- Projections

These calculations must always produce the same output for the same input.

---

## AI Explains, It Does Not Calculate

AI is responsible for explaining data.

AI is never responsible for mathematical calculations.

# Decision Framework

Every new feature proposed for Project Battle must answer the following questions before it is accepted.

## Question 1

Does this feature help the user understand their own data better?

If NO → Reject.

---

## Question 2

Can this feature be supported with evidence?

If NO → Reject.

---

## Question 3

Does this increase intelligence instead of complexity?

If NO → Reject.

---

## Question 4

Can this feature be explained transparently?

If NO → Reject.

---

## Question 5

Does this respect the philosophy:

"The app doesn't tell me what to do. It tells me what my data says."

If NO → Reject.

---

# Definition of Success

Project Battle is successful if:

- The user logs data consistently.
- The system produces trustworthy insights.
- Every conclusion is backed by evidence.
- The application remains simple.
- The user understands their own progress better than before.

Project Battle is NOT measured by:

- Number of features
- AI usage
- Visual effects
- Gamification
- Social engagement

It is measured by one question:

> "Did the application help the user understand what their own data was saying?"
